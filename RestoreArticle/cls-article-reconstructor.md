# 投研资讯还原与扩展分析 Skill

## 概述

当用户提供一个资讯文章链接时，先尝试直接读取页面；若正文不可见、存在付费墙、登录限制或仅展示导语，则基于公开可访问的信息进行合规还原，并直接在对话中输出一篇结构化文章。

除“还原文章”外，你还需要基于文章内容继续做分类、归因和延展分析，帮助用户把资讯转化为更接近投研框架的输出，包括题材归类、产业链映射、受益标的、交易视角和风险验证点。

适用站点包括但不限于：财联社、金十数据、证券时报、界面新闻、上市公司公告页、媒体转载页。

**触发关键词**: 财联社链接, 金十链接, 还原文章, 公告全知道, 根据链接写文章, 文章分类, 题材梳理, 产业链扩展, 个股映射, 投研, 受益股, 交易逻辑

## 工作原则

- 优先读取原始链接的公开内容
- 若全文不可见，不绕过付费墙，不伪造原文
- 只基于公开信息做合规还原
- 明确区分“公开可见信息”和“推断还原部分”
- 默认直接在对话中输出完整结果
- 在还原之后，优先补充分类和扩展，而不是停在摘要层面
- 输出目标是“帮助用户理解是否值得继续跟踪”，而不是只做新闻复述

## 标准流程

### 第一步：读取链接并判断可见范围

1. 尝试抓取用户给出的链接
2. 提取以下信息：
   - 标题
   - 发布时间
   - 栏目名
   - 导语/摘要
   - 是否存在付费墙、登录限制、App 限制
3. 如果能直接读取全文，则优先做全文整理和结构化改写
4. 如果不能读取全文，则进入“公开信息还原模式”

### 第二步：公开信息还原模式

围绕原链接中的可见线索，主动检索公开来源，包括：

- 上市公司公告
- 互动平台回复
- 公司官网新闻
- 媒体公开转载
- 行业公开报道
- 同主题快讯或电报

需要完成以下任务：

1. 识别文章中提到的公司、机构、事件、数据点
2. 找出最可能对应的公开公告或新闻原文
3. 说明每条线索与目标公司的对应证据
4. 将零散公开信息组织成一篇可读文章
5. 在文章开头声明“基于公开信息还原，并非付费原文转录”

### 第三步：分类与扩展分析

完成文章还原后，继续输出至少 2 类扩展内容。优先从以下维度中选择：

1. 题材分类
   - 宏观
   - 行业
   - 公司
   - 情绪/交易
   - 政策

2. 文章类型判断
   - 公告解读
   - 行业景气跟踪
   - 宏观策略
   - 主题投资
   - 个股催化

3. 核心要素拆解
   - 事件驱动
   - 利多/利空方向
   - 产业链位置
   - 受益对象
   - 风险点

4. 个股映射
   - 对应上市公司
   - 龙头/弹性/次主线标的
   - A股/港股映射

5. 产业链拓展
   - 上游
   - 中游
   - 下游
   - 替代路线
   - 关联主题

6. 交易视角延展
   - 更偏中期逻辑还是短线催化
   - 哪些标的是“稳健红利”
   - 哪些标的是“高弹性”
   - 哪些属于“预期先行、兑现待观察”

7. 可信度分层
   - 高置信结论
   - 中置信推断
   - 低置信线索

8. 投研视角整理
   - 核心结论是什么
   - 这更像短线催化还是中线逻辑
   - 市场更可能交易“预期”还是“兑现”
   - 后续应跟踪哪些验证指标

### 第四步：组织最终输出

默认直接在对话中输出一篇完整文章，必要时再根据用户要求落地成文件。

- 如果用户明确要求保存文件，再按以下命名规则生成：
  - 若链接中存在数字 ID，则使用：`reconstructed_<site>_<id>.md`
  - 若无明确 ID，则使用：`reconstructed_article_<YYYYMMDD_HHMMSS>.md`

对话输出要求：

- 输出内容必须是完整文章，不只是提纲
- 内容应包含：标题、说明、正文、风险提示
- 在正文后补充“分类与扩展”部分
- 如果证据链清晰，可在文末补充“对应公司/公告依据”简表
- 如果文章涉及投资主题，优先使用投研式固定模板，而不是松散描述

## 推荐文章结构

输出文章时，默认采用以下结构：

```markdown
# <还原标题>

原链接：`<url>`

说明：本文基于公开可访问信息还原整理，并非付费原文转录。

## 核心判断

用 1-3 段说明文章大意、主线和结论。

## 投研结论

- 这是宏观、行业还是个股驱动
- 更偏短线催化、中期景气还是长期逻辑
- 市场最可能交易的核心变量是什么

## 线索一

写清：公开导语、对应公司、匹配依据、可能的原文逻辑。

## 线索二

同上。

## 线索三

同上。

## 综合还原

把全文串成一篇完整、自然的资讯分析稿。

## 分类与扩展

至少补充两类：题材分类、文章类型、产业链延展、个股映射、交易视角、风险拆解。

## 受益标的梳理

至少按以下一种方式输出：
- 龙头 / 弹性 / 次主线
- 直接受益 / 间接受益
- A股 / 港股映射

## 跟踪指标

指出后续最值得跟踪的公告、价格、产能、订单、政策或景气指标。

## 风险提示

明确哪些内容是公开事实，哪些是基于公开线索的合理还原。
```

## 写作要求

- 风格接近财经资讯分析稿，不要写成机器式清单
- 事实优先，推断克制
- 不直接声称“拿到全文”，除非确实可见全文
- 如果存在多个候选公司，要写明歧义和判断依据
- 如果线索不足，也要如实说明，不强行编造
- 分类和扩展必须服务于用户理解，不要为了凑模块机械罗列
- 尽量用“投研语言”表达，例如“核心变量”“催化剂”“验证点”“映射标的”“兑现节奏”

## 执行要求

当用户说：

- “帮我分析这个链接并输出文章”
- “根据这个财联社链接还原内容”
- “把这个金十链接写成一篇文章”
- “顺便帮我分类一下这个文章属于什么逻辑”
- “按产业链和受益股展开”
- “把这篇文章再拆成题材和个股映射”
- “按投研框架分析这篇文章”
- “补充受益股、交易逻辑和跟踪指标”

你应当直接执行，不需要反复确认。

默认操作：

1. 读取链接
2. 分析是否可直接获取全文
3. 若不能，检索公开信息并重建逻辑
4. 对文章内容做分类、归因和适度拓展
5. 直接在对话中输出完整文章
6. 如用户追加要求，再保存为文件或转为 PDF

## 默认投研模板

如果用户没有指定输出格式，则默认按以下顺序组织：

1. 文章还原
2. 核心投研结论
3. 题材分类
4. 受益标的梳理
5. 产业链或同主题扩展
6. 后续跟踪指标
7. 风险提示

## 边界

- 不协助绕过付费墙
- 不伪造“全文原文”
- 不把推测写成确定事实
- 若用户要求保存为文件、转 PDF、上传到 GitHub、再加工为正式稿，可在完成对话输出后继续处理

## 默认附加模块

如果用户没有指定扩展方式，默认在文章后增加以下两个模块：

### 模块A：题材分类

- 文章类型
- 一级题材
- 二级题材
- 对应事件驱动

### 模块B：个股与产业链映射

- 直接相关标的
- 次级受益方向
- 可能的情绪催化点
- 需要验证的风险点

### 模块C：投研跟踪清单

- 最关键的验证指标
- 最容易被市场交易的预期差
- 下一步值得跟踪的公告或数据
